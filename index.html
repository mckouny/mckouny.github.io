Profiling users with .bash_history

Can we be identified using our bash behaviour?

Cyber attribution is hard. By analyzing the patterns in attacker's commands, we may be able to link different attacks to him, or at least to someone who had the same training.

For my dissertation, I collect (anonymized) .bash_history files from many users and use some ML™ to link them together. For that, I need a lot of data and I would appreciate if you shared your history with me.

To see how I handle your data securely and anonymously, read on.

How it works

Bash logs can contain sensitive information such as filenames or passwords. To protect your privacy, all files are processed using an anonymizing script. In short, it goes through the file word by word and only keeps the string if it is:

  • on a list of ~250 common, mostly built-in commands or operators ls sudo ifconfig vim | >
  • recognized as a short option/flag – a hyphen followed by 1 to 4 alphanumeric characters -p -alt -rf
  • regexxed as a long option/flag – double hyphen followed by alphanumeric strings, which are divided by another hyphen --follow-symlinks --help --version

Everything else gets hashed. This way I can still see some patterns while knowing very little about you.

In the next step, you can download the 30-line python script and run it locally before uploading the anonymized file. If you a) trust me b) can’t be bothered c) forget to hash your file, don’t worry – all files are processed using the same script (this time in JS) before they are sent over.

Submit your files

On most UNIX systems, the .bash_history file is in the home directory (~/.bash_history). As the default size is mere 500 lines, please try to upload multiple files (different machines, timeframes, as long as it’s all you). If you are by any chance authorized to upload logs of other users (!), upload each of them separately.

This website is hosted on github pages, so you can inspect the source code. Once you upload the file, it will be anonymized in your browser. You will see the result and then decide whether you want to submit it or not.

For added security, download the anonymizing script in Python and upload already anonymized files.
